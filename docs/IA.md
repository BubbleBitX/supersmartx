# Information Architecture

## Primary navigation
- `Dashboard`: recent activity and next action
- `Create`: build a new milestone post
- `Timeline`: saved outputs and reuse entry point
- `Profile`: saved identity and defaults
- `Templates`: selectable visual starting points
- `Pricing`: plan explanation and upgrade contact path

## Rules
- Navigation depth stays within 3 levels
- `Create` is the primary action from every app surface
- `Profile` is a supporting surface, not a required blocker
- `Timeline` is for reuse, not archive management complexity

## Page responsibilities
- `Dashboard`: orient and route the user to create or reuse
- `Create`: collect milestone input, preview output, export result
- `Timeline`: reopen prior entries and duplicate them into create flow
- `Profile`: maintain reusable name, role, company, and photo defaults
- `Templates`: choose a base layout without forcing template jargon
- `Pricing`: explain limits and route upgrade interest safely
