# Design System

## Principles
- Keep copy short and task-driven
- Use strong visual hierarchy
- Use one primary action per surface
- Prefer reuse of existing visual language over adding more variants

## Tokens
- `Color.*`
- `Spacing.*`
- `Radius.*`
- `Elevation.*`
- `Typography.*`
- `Motion.*`

## Current launch constraint
Large parts of the UI still use inline styles. Before scale-out, shared tokens should be moved into reusable primitives and theme variables.
